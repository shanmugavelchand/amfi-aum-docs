amfi_aum

A Python module for fetching Average Assets Under Management (AAUM) data from the Association of Mutual Funds in India (AMFI).

The module wraps the undocumented JSON endpoints used by amfiindia.com/aum-data/average-aum. It handles discovery of financial years and periods, downloads the data, and returns clean pandas.DataFrame objects that you can analyse or export.

Data is available from April 2006 to the present. From December 2010 onwards the data is published quarterly; before that it was monthly.

There are three data modes:

ModeDescriptionRows per call
FundwiseOne row per mutual fund house~52
Schemewise CategorywiseOne row per scheme, grouped by category~1,000+
Schemewise TypewiseOne row per scheme, grouped by type~1,000+

Installation

The module is a single file. Copy amfi_aum.py into your project directory. It requires three third-party packages: 

pip install requests pandas openpyxl

Python 3.10 or later is required.

Quick Start

List financial years and periods

import amfi_aum

years   = amfi_aum.get_financial_years()
periods = amfi_aum.get_periods(fy_id=1)

print(years[0])
# {"id": 1, "financial_year": "April 2025 - March 2026"}

print(periods[0])
# {"id": 1, "period": "January - March 2026"}

Fetch fund-level data for one period

df = amfi_aum.get_fundwise_data(fy_id=1, period_id=1)
print(df[["mutual_fund_name", "avg_aum_excl_domestic_fof_incl_overseas"]])

Fetch scheme-level data for one period

df = amfi_aum.get_schemewise_data(
    fy_id=1,
    period_id=1,
    str_type="Categorywise"
)
print(df["scheme_category"].unique())

Download everything and save to files

dfs = amfi_aum.scrape_all(output_dir="./output")

# dfs is a dict with three DataFrames:
print(dfs["fundwise"].shape)
print(dfs["schemewise_category"].shape)
print(dfs["schemewise_type"].shape)
scrape_all() makes approximately 240 HTTP requests. With the default 0.5-second delay it takes around 4 to 5 minutes to finish.

get_financial_years()

get_financial_years()

Returns a list of all financial years for which AMFI has published average AUM data. Use the id from each entry when calling get_periods(), get_fundwise_data(), or get_schemewise_data().

Parameters

This function takes no parameters.

Returns

list of dict list

KeyTypeDescription
idintNumeric identifier. Pass this as fy_id in other functions.
financial_yearstrHuman-readable label, e.g. "April 2025 - March 2026".

Example

years = amfi_aum.get_financial_years()

for y in years:
    print(y["id"], y["financial_year"])

# 1  April 2025 - March 2026
# 2  April 2024 - March 2025
# ...
# 20 April 2006 - March 2007

get_periods()

get_periods(fy_id, mode="fundwise", str_type="Categorywise")

Returns a list of reporting periods available for a given financial year. Use the id from each entry as period_id when calling get_fundwise_data() or get_schemewise_data().

Parameters

NameTypeDescription
fy_id int Financial year ID from get_financial_years().
mode optional str Which endpoint to query. Use "fundwise" (default) when you intend to call get_fundwise_data(), or "schemewise" when you intend to call get_schemewise_data().
str_type optional str Only relevant when mode="schemewise". Either "Categorywise" (default) or "Typewise".

Returns

list of dict

KeyTypeDescription
idintNumeric identifier. Pass this as period_id in data-fetch functions.
periodstrHuman-readable label, e.g. "January - March 2026".

Example

# Periods for use with get_fundwise_data()
periods = amfi_aum.get_periods(fy_id=1)

# Periods for use with get_schemewise_data(str_type="Typewise")
periods = amfi_aum.get_periods(fy_id=1, mode="schemewise", str_type="Typewise")

get_fundwise_data()

get_fundwise_data(fy_id, period_id)

Fetches fund-house-level average AUM for one financial year and period. Returns one row per mutual fund house plus a final "Grand Total" row.

Parameters

NameTypeDescription
fy_id int Financial year ID from get_financial_years().
period_id int Period ID from get_periods().

Returns

pandas.DataFrame

  • financial_yearstrLabel for the financial year.
  • periodstrLabel for the reporting period.
  • sr_nostrSerial number. "TOTAL" on the grand total row.
  • mutual_fund_namestrName of the fund house.
  • avg_aum_excl_domestic_fof_incl_overseasfloatAverage AUM excluding domestic fund-of-funds but including overseas fund-of-funds. In INR lakhs.
  • avg_aum_fof_domesticfloatAverage AUM from domestic fund-of-funds only. In INR lakhs.

Example

df = amfi_aum.get_fundwise_data(fy_id=1, period_id=1)
print(df.shape)           # (53, 6) for ~52 funds + 1 total row
print(df.columns.tolist())

get_schemewise_data()

get_schemewise_data(fy_id, period_id, str_type="Categorywise", mf_id=0)

Fetches scheme-level average AUM for one financial year, period, and grouping type. Returns one row per mutual fund scheme. Rows where scheme_name is "[FUND TOTAL]" are fund-house-level subtotals inserted for convenience.

Parameters

NameTypeDescription
fy_id int Financial year ID from get_financial_years().
period_id int Period ID from get_periods(mode="schemewise").
str_type optional str How schemes are grouped. Either "Categorywise" (default) or "Typewise".
Use amfi_aum.SCHEMEWISE_TYPES to get both values as a tuple.
mf_id optional int Fund house filter. 0 (default) returns all fund houses in a single response. Pass a specific strMFId value from a prior response to filter to one fund house.

Returns

pandas.DataFrame

  • financial_yearstrLabel for the financial year.
  • periodstrLabel for the reporting period.
  • typestr"Categorywise" or "Typewise".
  • mf_idstrFund house identifier as returned by AMFI.
  • mutual_fund_namestrName of the fund house.
  • scheme_categorystrScheme category description.
  • scheme_namestrFull scheme name as registered with AMFI. "[FUND TOTAL]" on subtotal rows.
  • amfi_codeintUnique AMFI scheme code. Null on subtotal rows.
  • avg_aum_excl_domestic_fof_incl_overseasfloatAverage AUM excluding domestic fund-of-funds but including overseas fund-of-funds. In INR lakhs.
  • avg_aum_fof_domesticfloatAverage AUM from domestic fund-of-funds only. In INR lakhs.

Example

# Fetch Typewise grouping
df = amfi_aum.get_schemewise_data(
    fy_id=1,
    period_id=1,
    str_type="Typewise"
)

# Filter out subtotal rows
schemes_only = df[df["scheme_name"] != "[FUND TOTAL]"]
print(schemes_only.shape)

scrape_all()

scrape_all(output_dir=None, delay=0.5)

Downloads all available AUM data across every financial year, period, and grouping mode. Returns the results as a dictionary of three DataFrames and optionally saves them to files.

This is the main entry point when you want a complete data extract. It iterates through all 20 financial years, all periods within each year, and all three modes (Fundwise, Schemewise Categorywise, Schemewise Typewise).

This function makes approximately 240 HTTP requests. With the default 0.5-second delay it takes around 4 to 5 minutes.

Parameters

NameTypeDescription
output_dir optional str Directory where CSV and Excel files will be saved. Pass None (default) to skip saving and only get the DataFrames back.
delay optional float Seconds to wait between HTTP requests. Default is 0.5. Increase this if the server returns errors.

Returns

dict with the following keys:

KeyValue
fundwiseDataFrame with fund-house-level data for all periods.
schemewise_categoryDataFrame with scheme-level Categorywise data for all periods.
schemewise_typeDataFrame with scheme-level Typewise data for all periods.

When output_dir is provided, the following files are also written:

FileContents
amfi_aum_fundwise.csvFundwise DataFrame as CSV.
amfi_aum_schemewise_category.csvSchemewise Categorywise DataFrame as CSV.
amfi_aum_schemewise_type.csvSchemewise Typewise DataFrame as CSV.
amfi_aum_all.xlsxAll three DataFrames as separate sheets in one workbook.

Example

# Download everything and save to ./output/
dfs = amfi_aum.scrape_all(output_dir="./output")

# Use the DataFrames without saving
dfs = amfi_aum.scrape_all()
fw  = dfs["fundwise"]
cat = dfs["schemewise_category"]
typ = dfs["schemewise_type"]

export_to_csv()

export_to_csv(dataframes, output_dir)

Saves a dictionary of DataFrames to individual CSV files. One CSV file is created per DataFrame. The output directory is created automatically if it does not exist.

Parameters

NameTypeDescription
dataframes dict A dictionary mapping a name string to a pandas.DataFrame. Typically the return value of scrape_all().
output_dir str Directory where files will be written.

Returns

list of str - the absolute paths of the files written.

Example

dfs   = amfi_aum.scrape_all()

ID Lookup

Financial year IDs and period IDs are assigned by AMFI and may change if AMFI adds historical data. Always call get_financial_years() and get_periods() at runtime rather than hard-coding IDs.

As of May 2026 the financial year IDs run from 1 (most recent, April 2025 - March 2026) to 20 (oldest, April 2006 - March 2007).

Period IDs within each financial year start at 1 for the most recent quarter and increment as you go backwards in time.

Iterating all combinations

import amfi_aum

for fy in amfi_aum.get_financial_years():
    periods = amfi_aum.get_periods(fy["id"])
    for p in periods:
        df = amfi_aum.get_fundwise_data(fy["id"], p["id"])
        print(fy["financial_year"], p["period"], df.shape)

Error Handling

All HTTP calls go through the internal _get() helper which retries up to 3 times with an exponential back-off (2, 4, 6 seconds). After all retries are exhausted it prints a warning and returns an empty dict. Data functions that receive an empty dict return an empty DataFrame.

scrape_all() raises a RuntimeError if the initial financial-years call fails, because nothing can proceed without it. All other failures produce empty DataFrames for the affected periods and are logged to stdout.

The AMFI API does not require authentication and has no published rate limit. A delay of 0.5 seconds between requests has been observed to be reliable. If you encounter HTTP 429 or 5xx errors, increase the delay parameter in scrape_all().
